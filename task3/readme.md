Принятые решения по Data Quality
Общий подход

Для упрощения дальнейшей загрузки в БД все входные данные были приведены к единому формату CSV.

При обнаружении некорректных значений записи не удалялись, а проблемные поля приводились к пустым значениям (NULL), а информация о проблеме сохранялась в лог-файл [problems.txt.](data/cleaned/problems.txt)

Такой подход позволяет сохранить максимальный объем данных и избежать потери потенциально полезной информации.
